Il DB SQL ha lo stesso contenuto informativo del PDF.
Il PDF è una estrazione di quelle tabelle
La tabella notifications ha la seguente struttura:

TABLE "notifications"

"id" INTEGER -> ordine arrivo notifiche
"type" INTEGER NOT NULL ->
"testID" TEXT NOT NULL,
"session" INTEGER NOT NULL,
"timestamp" DATETIME NOT NULL,
"testStatus" INTEGER NOT NULL,
"testResult" INTEGER NOT NULL,
"error" TEXT NOT NULL,
"updateType" TEXT NOT NULL,
"updateValue" TEXT NOT NULL,

"report" TEXT NOT NULL,

id: intero che rappresenta l'ordine di arrivo della notifica; è la chiave primaria con incremento automatico.
type: intero che indica il tipo di notifica, mappato all'enum NotificationType (es. 0 = Unknown, 1 = Init, ..., 7 = Error).
testID: stringa che identifica in modo univoco il test a cui la notifica si riferisce. Si traduce tramite altra tabella, dove per ogni testID trovi la relativa label.
session: intero che rappresenta l'ID della sessione del test o processo. Quante volte esegui il test.
timestamp: data e ora in cui è stata generata la notifica.
testStatus: intero che rappresenta lo stato attuale del test, mappato all'enum Status_t (es. 2 = Running, 3 = Terminated, ecc.).
testResult: intero che rappresenta l'esito finale del test, mappato all'enum Result_t (es. 2 = Successful, 3 = Failed).
error: testo libero che descrive un eventuale errore verificatosi; obbligatorio anche se può essere vuoto.
report: contenuto testuale del report o log relativo alla notifica
A seguire gli enumerati.


enum NotificationType {Unknown = 0, Init = 1, Running = 2, Update = 3, 
			Terminated = 4, Stopped = 5, Deleted = 6, Error = 7};

    enum Result_t {
        Default = 0,
    NotExecuted = 1,
        Successful = 2,
        Failed = 3
    };   

    enum Status_t {
        Default = 0,
        Init = 1,
        Running = 2,
        Terminated = 3,
        Stopped = 4,
        Deleted = 5,
        Error = 6
    };


La ripetizione del testId nella tabella è dovuta al fatto che HMI e software di diagnostica si scambiano più messaggi, non solo il risultato per eseguire un test.

Se hai notato tutti i report/risultati sono presenti sulle notifiche di tipo 4. 

La notifica di tipo 4 se la mappi sull'enumerato  NotificationType, corrisponde alla notifica che il sw di diagnostica invia quando ha concluso il test.
